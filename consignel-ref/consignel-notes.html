Sécurisation informatique

Codage et sécurisation

Les données faisant référence au compte et transitant par l'internet sont chiffrées. Le nombre transféré pour L'identifiant de l'utilisateur est la somme du code numérique de chaque caractère multiplié par un nombre dépendant de son rang dans la chaîne de caractères (codecar*1+codecar*11+codecar*21+codecar*31,etc.). Seules les empreintes chiffrées sont transférées. La vérification se fait sur les empreintes chiffrées calculées indépendamment côté utilisateur et côté serveur.

Les chiffres sont modifiés par un numéro de session aléatoire fourni par le serveur. Le chiffrage évoluera en fonction de la consommation des ordinateurs et des progrès des algorithmes.

Toutes les informations stockées sont soumises à un nettoyage de sécurité (regex d'élimination des caractères ayant un effet de programmation) et de précaution (double vérification). Les données sont séparées en plusieurs fichiers.

Les données entre l'utilisateur et le serveur sont transmises en https.

Les répertoires du serveur sont protégés par .htacces

Options All -Indexes
<files .baseconsignel*>
order allow,deny

</files>
<files *.json>

Un format ouvert json ou autre format texte simple est utilisé pour les données d'intérêt commun accessibles à tous.

Javascript est utilisé sur le navigateur de l'utilisateur pour faire le codage des informations avant l'envoi ou l'utilisation par l'application.

Le stockage des données non nominatives dans la mémoire locale du navigateur (localstorage) est soumis à l'autorisation de l'utilisateur dans les préférences.

Préférences

Stockage local

Données dans l'ordinateur ou le téléphone

Préférences générales du stockage

Localstorage stocke les données de préférences, les fichiers publics et les fichiers personnels selon les autorisations accordées.

Au démarrage le fichier lit les préférences d'autorisations

"consignel":[0,0,0]

Par défaut, l'enregistrement dans localstorage est interdit. Pour l'usage facile sur sa machine personnelle, il est conseillé d'autoriser les 3 l'enregistrement des préférences , l'enregistrement des fichiers publics et l'enregistrement des fichiers personnels .

L'utilisateur peut interdire les trois depuis les préférences de son navigateur. Il peut aussi le faire depuis les préférences du ↺onsignél. Lorsque l'autorisation est retirée c'est fait par localStorage.removeItem("consignel");. L'interdiction n'efface pas les caches du navigateur.

Les fichiers publics sont des listes pour aider à faire les entrées

"cherchequoimini":["activite1","activite2",....,"activite3"]

"cherchefaire":["verbe1","verbe2",....,"verbe3"]

"cherchequoi":["quoi1","quoi2",....,"quoi3"]

"chercheparqui":["parqui1","parqui2",....,"parqui3"]

"cherchepourqui":["pourqui1","pourqui2",....,"pourqui3"]

Ces fichiers ne sont pas cryptés

Les fichiers personnels sont des tableaux avec des valeurs d'activité. Voir le codage dans les formats de données

Par défaut, il n'y a que des nombres sous forme de chaînes dans les tableaux, même s'il représentent des noms ou des dates.

Les données sont fragmentées pour éviter les correspondances en lecture directe. Les noms sont chiffrés.

Dossier et nom de fichier des données de l'utilisateur

Les données sont dispersées dans plusieurs fichiers

Le chiffre de l'utilisateur donne aussi le chemin d'accès au dossier de l'utilisateur par sous-dossiers contenant un maximum de 100 sous-dossiers.

Par exemple le numéro de l'utilisateur consignel 39382 sera stocké dans le dossier .consignel-base/82/93/3/ sur le serveur son identification pour localstorage commencera par u suivi du code combiné de l'utilisateur et de son mot de passe

Le dossier contient les fichiers personnels de l'utilisateur. Sur le serveur, il est protégés par .htaccess. code activité: spécifique au ↺onsignel. Le code activité du ↺onsignel est calculé par chiffrage de la description de l'élément précédée de act. (act249642) Il ne tient pas compte des quantités et des valeurs. Le code activité d'une activité particulière (act249642p23983274) contient un deuxième code incluant la quantité, l'unité et les valeurs. Pour les correspondances avec d'autres classification, il y a des fichiers de traduction. Les catégories sont des éléments.
q: quantité
u: unité pour les quantités
↺: Valeur de compensation en ↺onsignel (montant dépendant du calcul du da↺ avec les 3 autres valeurs)
$: Valeur marchande en $ (1↺onsignel=1$CAN au Canada) (montant lié à la quantité)
*E: Valeur environnementale en McaEE (montant lié à la quantité)
durée: Quantité en heures décimales
*S: Valeur sociale (montant lié à la durée)
Fois/an: Récurrence

Toutes les valeurs sont relatives à l'unité choisie, même la durée.
Pour le total, la valeur marchande et la valeur environnementale sont multiplées par la quantité. La valeur sociale est multipliée par la quantité et par la durée.
