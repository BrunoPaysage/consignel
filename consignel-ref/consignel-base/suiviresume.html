00000-suiviresume.json : Historique des résumés du compte

Références/consignel-base/00/suiviresume

Le script php écrit ce fichier pour chaque proposition et acceptation de transaction. Il le lit sur demande de l'utilisateur. Le script php renvoie le contenu de l'historique au demandeur autorisé. Ce fichier doit être autorisé par l'utilisateur par un nombre de jours de stockage dans ses préférences. Une durée de 0 jours efface le fichier s'il existe lors de la transaction suivante ou de l'actualisation de la page.

Utilisateur, codé, base utilisateur

Le nom du fichier se lit
00000-suiviresume.json : identifiant de l'utilisateur
00000-suiviresume.json : identifiant de la nature du fichier
00000-suiviresume.json : extension du fichier simplifiant la protection .htaccess.
Le contenu des enregistrements successifs dans le fichier est du json incomplet crypté. Le format incomplet est suffisant pour le script php.

Le fichier contient:
encode pour stokage "transaction",1,2,3,4
encode pour stokage "transaction",1,2,3,4
etc.

Les éléments des enregistrements sont les suivants:

  • "transaction" : Nom du fichier de la proposition de transaction "tra00000000_0000_0000000" sans l'extension ".json". On peut aussi avoir le suivi acc, ann, exp. Le mom de la transaction contient la date de la transaction
  • 1 : Solde du compte en ↺
  • 2 : Minimum récent en ↺
  • 3 : Dépense autorisée par jour en ↺
  • 4 : Maximum récent en ↺
