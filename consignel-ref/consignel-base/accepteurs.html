00000-acepteurs.json : dates d'acceptation et accepteurs de l'année

Références/consignel-base/00/accepteurs

Le script php écrit ce fichier pour chaque proposition accepté de transaction sans redondance. Il le lit pour vérifier si l'utilisateur a le droit de faire l'ouverture d'un compte. Ce fichier est indispensable pour établir la toile de confiance du système.

Système, codé, base de l'utilisateur

Le nom du fichier se lit
00000-accepteurs.json : identifiant de l'utilisateur
00000-accepteurs.json : identifiant de la nature du fichier
00000-accepteurs.json : extension du fichier simplifiant la protection .htaccess.
Le contenu est un fichier json crypté.

Le fichier contient:
encode pour stokage ["id1":"numjour",""id2":"numjour",...]

Les éléments des enregistrements sont les suivants:

  • "id1" : "Numéro interne d'identification de la personne qui accepte"
  • "numjour" : "Numéro du jour de l'acceptation commençant par les 4 chiffres de l'année suivi des 3 chiffres du numéro de jour dans l'année"
