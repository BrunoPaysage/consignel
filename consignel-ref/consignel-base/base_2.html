consignel-base/2/ : Base des données de suivi des transactions

Références/consignel-base/2/suivi des transactions

Le script php utilise cette base pour servir aux utilisateurs autorisés les transactions offertes par les autres utilisateurs et en faire le suivi.

Système, codé, base des transactions, accès au demandeur selon autorisations du proposeur ou du destinataire de la transaction

Cette base est organisée par date.
2/année/mois/jour/heure/
année: 4 chiffres, mois: 2 chiffres, jour: 2 chiffres, heure: 2 chiffres

Le répertoire de l'heure contient les fichiers permettant le suivi des transactions pour la date et l'heure indiquée

  • Suivi des transactions tra00000000_00-suivi.json Un seul fichier pour toutes les transactions à cette heure
  • Proposition de transaction tra00000000_0000_0000000.json Un fichier par proposition
  • Transaction acceptée acc00000000_0000_0000000.json Un fichier d'acceptation par proposition
  • Transaction annulée ann00000000_0000_0000000.json Un fichier d'annulation par proposition
  • Transaction expirée exp00000000_0000_0000000.json Un fichier d'expiration par proposition

Les numéros sont les mêmes pour le suivi de la transaction dans ses états possibles: proposée, acceptée annulée ou expirée. Ces fichiers ne sont pas modifiés après leur inscription initiale.
