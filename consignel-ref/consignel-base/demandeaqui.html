00000-resume2dates.json : 2 derniers résumés du compte

Références/consignel-base/00/suivi31jours

Le script php écrit ce fichier pour chaque proposition accepté de transaction sans redondance. Il le lit sur demande de l'utilisateur pour lui fournir une liste de personnes à qui il offre des transactions qui sont acceptées ou des personnes qui lui on fait une offre qu'il a accepté.

Le nom du fichier se lit
00000-demandeaqui.json : identifiant de l'utilisateur
00000demandeaqui.json : identifiant de la nature du fichier
00000-demandeaqui.json : extension du fichier simplifiant la protection .htaccess.
Le contenu est un fichier json crypté.

Le fichier contient:
encode pour stokage ["pseudo1","pseudo2","pseudo3"]

Les éléments des enregistrements sont les suivants:

  • "pseudo1" : "Nom public ou pseudonyme d'un autre utilisateur"
  • "pseudo2" : "Nom public ou pseudonyme d'un autre utilisateur"
  • "pseudo3" : "Nom public ou pseudonyme d'un autre utilisateur"
