00000-resume2dates.json : 2 derniers résumés du compte

Références/consignel-base/00/suivi31jours

Le script php écrit ce fichier pour chaque proposition et acceptation de transaction. Il le lit pour déterminer des écarts de dates.

Le nom du fichier se lit
00000-resume2dates.json : identifiant de l'utilisateur
00000-resume2dates.json : identifiant de la nature du fichier
00000-resume2dates.json : extension du fichier simplifiant la protection .htaccess.
Le contenu est un fichier json crypté.

Le fichier contient:
encode pour stokage {"3":"acc20190321_1357_1646152","0":"acc20190806_1722_1858852","1":"dac20190806_1730_3535","4":138,"5":0,"6":"79","7":"217","9":"217","8":"217"}

Les éléments des enregistrements sont les suivants:

  • "0" : "Identifiant de la transaction précédente"
  • "1" : "Identification de la transaction en cours"
  • "2" :
  • "3" : "Identification de la première transaction de l'utilisateur"
  • "4" : ancienneté en nombre de jours
  • "5" : écart d'année entre le deux transaction. 0 si c'est la même année
  • "6" : numéro du jour de la première inscription
  • "7" : numéro du jour de la transaction en cours à partir de la date du fichier transmis
  • "8" : numéro du jour de la transaction précédente à partir de la lecture de ce fichier avant modification
  • "9" : numéro du jour de la transaction en cours à partir de l'horloge du serveur
