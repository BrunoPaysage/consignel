acc00000000_0000_0000000.json : transaction acceptée

Références/consignel-base/2/transaction acceptée

Permet au script php de savoir si la transaction a déjà été acceptée, par qui et quand

Le nom du fichier se lit

  • acc00000000_0000_0000000.json : identifiant d'acceptation de proposition
  • acc00000000_0000_0000000.json : identifiant de date de la proposition de transaction. Année en 4 chiffres, mois en 2 chiffres et jour en 2 chiffres
  • acc00000000_0000_0000000.json : identifiant d'heure et la minute de la proposition de transaction. 2 chiffres chaque
  • acc00000000_0000_0000000.json : code de la transaction en lien avec son contenu. Nombre de chiffres variables
  • acc00000000_0000_0000000.json : extension du fichier simplifiant la protection .htaccess.
    La combinaison de la date et du codage du contenu permet d'identifier de façon unique une transaction.

encode pour stokage "identifiant","offre","demande","momentaccepte","codeaccepteur","durée","↺","codeproposeur"

  • "identifiant" : Le même que le nom de fichier "acc00000000_0000_0000000" sans l'extension. Le même que l'identifiant de la proposition de transaction correspondante avec "acc" au lieu de "tra" comme préfixe
  • "offre" : Code de l'offre dans la proposition de transaction correspondante "0000000" Nombre de chiffres variable correspondant au contenu du groupe offre
  • "demande" : Code de la demande dans la proposition de transaction correspondante "0000000" Nombre de chiffres variable correspondant au contenu du groupe demande
  • "momentaccepte" : Date et heure du moment de l'acceptation "00000000_0000" 24 chiffres pour l'année, 2 pour le mois, 2 pour le jour _ 2 pour l'heure et 2 pour les minutes. Le moment de la proposition de transaction est contenu dans l'identifiant.
  • "codeaccepteur" : Code de l'accepteur
  • "durée" : Nombre de jour de validité de la proposition initiale
  • "↺" : Montant de ↺onsignél demandé dans la proposition initiale
  • "codeproposeur" : Code du proposeur de la transaction
