00000-gain365jours.json : Historique des revenus et dépenses journaliers

Références/consignel-base/00/suivi31jours

Le script php écrit ce fichier pour chaque proposition et acceptation de transaction. Il le lit pour déterminer le minimum et le maximum récent du compte. Seuls les 31 jours précédant la date de l'enregistrement sont conservés, les autres sont éliminés.

Systéme, codé, base utilisateur

Le nom du fichier se lit
00000-gain365jours.json : identifiant de l'utilisateur
00000-gain365jours.json : identifiant de la nature du fichier
00000-gain365jours.json : extension du fichier simplifiant la protection .htaccess.
Le contenu est un fichier json crypté.

Le fichier contient:
encode pour stokage { "id":valeur, "id":valeur, "id":valeur }
etc.

Les éléments des enregistrements sont les suivants:

  • "id" : le numéro du jour de l'année exemple "35" ou "147"
  • valeur : Le cummul des gains du jour en ↺ exemple 23 ou 427.25
