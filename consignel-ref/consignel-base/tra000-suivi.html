tra000-suivi.html : Suivi des transactions pour l'heure en cours

Références/consignel-base/2/suivi transactions

Le script php écrit ce fichier pour chaque proposition et acceptation de transaction. Il le lit pour chaque demande d'acceptation de transaction. Le script php renvoie l'information le code de suivi accompagné du contenu de la proposition au demandeur autorisé.

Le nom du fichier se lit
tra00000000_00-suivi.json : identifiant de proposition
tra00000000_00-suivi.json : identifiant de date. Année en 4 chiffres, mois en 2 chiffres et jour en 2 chiffres
tra00000000_00-suivi.json : identifiant d'heure des transactions en 2 chiffres
tra00000000_00-suivi.json : identifiant de la nature du fichier
tra00000000_00-suivi.json : extension du fichier simplifiant la protection .htaccess.
Le contenu des enregistrements successifs dans le fichier est du json incomplet crypté. Le format incomplet est suffisant pour le script php.

Le fichier contient:
encode pour stokage "transaction","offre","demande","moment","pourqui","duréevalide","montant↺demandé","dequi"
encode pour stokage "transaction","offre","demande","moment","pourqui","duréevalide","montant↺demandé","dequi"
etc.

Les éléments des enregistrements sont les suivants:

  • "transaction" : Nom du fichier de la proposition de transaction "tra00000000_0000_0000000" sans l'extension ".json".
    "tra" Préfixe pour une proposition.
    "acc" Préfixe pour une acceptation de proposition.
    "ann" Préfixe pour une annulation de proposition.
    Le nom comporte le moment de la proposition initiale dans les deux premières séries de chiffres et la nature de la transaction dans la dernière série de chiffres.
  • "offre" : Code de l'offre. "00000" nombre de chiffres variable
  • "demande" : Code de la demande. "00000" nombre de chiffres variables
  • "moment" : Identifiant de date. "00000000_0000" année en 4 chiffres, mois en 2 chiffres et jour en 2 chiffres souligné heure et minutes 2 chiffres chacun. Moment de la proposition, de l'acceptation ou de l'annulation selon le préfixe de la transaction.
  • "pourqui" : Code pour restreindre l'acceptation de la proposition à un utilisateur pour un identifiant commençant par "tra". "00000" nombre de chiffres variable, 0 pour ouvrir la proposition à tous. Code du proposeur initial pour un identifiant commençant par "acc". Code du proposeur initial pour un identifiant commençant par "ann".
  • "duréevalide" : Nombre de jours de validité de la proposition "000" maximum 365
  • "montant↺demandé" : Montant de ↺onsignel demandés "000.00" accepte les nombres décimaux mais inscription sous forme de chaîne
  • "dequi" : Code du proposeur initial pour les préfixes "tra", "ann". Code de l'accepteur pour le prefixe "acc". "00000" nombre de chiffres variable.
